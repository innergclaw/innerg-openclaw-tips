Mindset & Approach

Evolution Favors Action
Approach building in a decisive, iterative way. Start with what you've always wanted to create. Experiment without expecting instant expertise. Give yourself time to improve. This mindset evolved InnerG Intel from an idea to a brand.
"Evolution favors action. Ship first, perfect later."
Resourceful Before Asking
Don't overthink. Figure it out first, come back with answers. Read the file, search context, use what you have. Mental notes don't survive restarts — WRITE IT DOWN. Dozens of projects built this way.
Move With Intention
Let go of perfectionism. Not all work needs to be exactly how you'd plan it. Every action has purpose. No half-measures. Critical for shipping AI-generated work fast.
"I ship work I don't overthink."

Coding & Workflow with AI Agents

Tip Explanation
Prefer zai-turbo & kimi For large codebases — fewer mistakes, less hand-holding, better long-running execution
Spawn Sub-Agents Flash models for research/drafting — save your brain for strategy. Queue multiple agents like a dev team.
Intentional Under-Prompting Leave gaps for creative solutions. Agents fill them better than you expect.
Close the Loop Agents should compile → lint → test → execute → self-verify via local checks
SKILL.md Everything If you do it twice, skill it. Everything should be a skill. MCP is a crutch.
Focus on Architecture Prioritize system design, modularity, outcomes over code details. Most code is boring.
Top Engineers Optimize For shipping real products. Quality over quantity, always.

Running & Optimizing OpenClaw

Area Tips
Tiered Model Routing Use cheap models for simple tasks; reserve best models for complex reasoning. Improves efficiency and cuts costs. Use /model for switching.
Persistence Store knowledge in USER.md, AGENTS.md, MEMORY.md — prevents relearning. MEMORY.md = long-term memory.
Cron vs Heartbeat Exact timing matters? Use cron. Batching similar checks? Use heartbeat. Background tasks, not sessions.
Guardrails Use SKILL.md files; add anti-loop rules, checkpoints, summaries. Close the loop.
One Workflow First Master a single use case before expanding — avoid confusion and failures.
Model Quality Performance depends on LLM reasoning + tool use. Don't blame the tool when it's the model.
